# Shuncode AI — Обзор возможностей

> **Shuncode AI** — AI-ассистент для разработки, встроенный в редактор кода на базе VS Code.
> Полный аналог Cursor AI с рядом уникальных преимуществ.

*Актуально на: 1 марта 2026*

---

## Что это?

Shuncode AI — это форк VS Code с **глубоко интегрированным AI-агентом**. В отличие от расширений (Copilot, Continue и пр.), AI здесь является частью самого редактора. Это даёт полный контроль над UX, производительностью и безопасностью.

---

## Ключевые возможности

### 🔧 Инструменты AI-агента (30+ штук)

| Что умеет | Описание |
|---|---|
| **Читать файлы** | Любой файл проекта, с номерами строк для точных ссылок |
| **Создавать и редактировать файлы** | Полное создание, замена блоков, патчи |
| **Выполнять команды** | Терминал: сборка, тесты, git, npm, Docker и пр. |
| **Семантический поиск** | Поиск по смыслу кода (embedding-индекс, 90 000+ чанков) |
| **Regex-поиск (ripgrep)** | Быстрый поиск по шаблонам |
| **Glob-поиск файлов** | Поиск по имени и расширению |
| **Web-поиск** | Поиск информации в интернете |
| **Браузер (Puppeteer)** | Автоматизация: скриншоты, клики, заполнение форм |
| **MCP-интеграции** | Подключение внешних сервисов (Context7, БД, API и др.) |
| **Диагностика (линтер)** | Чтение ошибок ESLint, TypeScript и пр. |
| **Удаление файлов** | Удаление с настраиваемым auto-approval (по умолчанию требует подтверждения) |
| **Мониторинг терминалов** | Видит открытые терминалы пользователя и их состояние |

### 🧠 4 режима работы

| Режим | Назначение |
|---|---|
| **Act** | Основной. Полный доступ к инструментам, модификация кода |
| **Plan** | Проектирование. Read-only, без изменений в коде |
| **Debug** | Отладка. Систематический анализ с runtime-данными |
| **Ask** | Вопрос-ответ. Исследование кодовой базы без изменений |

Модель может **динамически переключаться** между режимами в процессе работы.

### 📝 Inline Diff — редактирование прямо в коде

- Изменения от AI показываются **прямо в редакторе** (зелёное — добавлено, красное — удалено)
- Кнопки **Accept / Reject** на каждый блок
- Горячие клавиши: `Alt+]` / `Alt+[` для навигации, `Ctrl+Shift+Y/N` для принятия/отклонения
- Панель **Pending Changes** с навигацией по всем незакоммиченным изменениям
- Автоматическое отслеживание позиций при дальнейшем редактировании

### 💬 Чат с AI

- **TurnBlock-архитектура** — сообщения сгруппированы по «ходам» (запрос → ответ)
- **Sticky-заголовки** — сообщение пользователя всегда видно при скролле
- **Сворачиваемый блок «Thinking»** — видно, что модель «думает»
- **EditCard** — карточки изменений с якорями на diff в редакторе
- **Edit & Resend** — редактирование и повторная отправка сообщений
- **Отображение ошибок API** — сбои соединения не скрываются

### 🔍 Семантический поиск

- **Embedding-индекс** всего проекта (90 000+ чанков)
- Поиск по смыслу: "где обрабатываются ошибки авторизации?" → точные результаты
- **Гибридный режим**: семантика + ключевые слова
- Индекс обновляется в фоне через Worker Threads

### 🌐 40+ API-провайдеров

Поддержка моделей от:
- **OpenAI** (GPT-4o, o1, o3) — дефолтный провайдер
- **Anthropic** (Claude Sonnet, Opus, Haiku)
- **Google** (Gemini 2.5 Pro/Flash)
- **GigaChat** (Сбер: GigaChat-2 Pro, GigaChat-2 Max) — нативные function calls
- **YandexGPT** (Яндекс: YandexGPT 5 Pro, YandexGPT 5 Lite)
- **Open-source** (Qwen, DeepSeek, Llama, Mistral и др.)
- **OpenRouter** (агрегатор 200+ моделей)
- Любой OpenAI-совместимый API (локальные модели через Ollama, LM Studio, vLLM)

### 🛡️ Адаптация под слабые модели

- **Lightweight Mode** — упрощённый промпт (XS-вариант) для слабых/бесплатных моделей
- Упрощённые инструменты редактирования (`delete_block`, `replace_text`)
- 11 вариантов промптов, оптимизированных под конкретные семейства моделей

### 🎤 Голосовой ввод

- **Офлайн распознавание речи** — работает без интернета
- Запись через микрофон → транскрипция → текст в чате
- 3 уровня качества: Быстрое / Стандарт / Высокое
- Поддержка 50+ языков (включая русский)
- Выбор микрофона: авто-определение или ручной выбор в настройках
- Список доступных устройств с пометкой системного микрофона
- Ручной выбор сохраняется между сессиями (авто-выбор не перезаписывает)
- Кнопка «Обновить» для актуализации списка устройств
- Настройки: секция «Голос» (язык, качество, микрофон, загрузка компонентов)
- Компоненты скачиваются с CDN при первом использовании (авторизация по JWT)
- Анимация записи: пульсирующий индикатор + таймер
- Воронка: неактивный микрофон в чате → настройки → авторизация → загрузка → готово
- Бесплатно: быстрое качество; Платно: стандарт + высокое

**Статус:** скрыт, код сохранён.

Провайдер `shuncode` в списке API-провайдеров — заготовка под собственный прокси моделей через OpenRouter (пользователь не вводит ключ, биллинг через Shuncode-аккаунт). Пока не реализован.

**Как вернуть:**
1. `webview-ui/src/utils/getConfiguredProviders.ts` — раскомментировать `configured.push("shuncode")`
2. `src/core/controller/index.ts` — удалить блок "Migrate away from deprecated shuncode provider" из конструктора (~строка 151)
3. `src/core/controller/index.ts` — в `handleAuthCallback()` вернуть установку `planModeApiProvider = "shuncode"` после авторизации

**Связанные файлы:**
- `webview-ui/src/components/settings/providers/ShuncodeProvider.tsx` — UI провайдера
- `src/core/api/providers/shuncode.ts` — API handler
- `src/shared/providers/providers.json` — shuncode НЕ в этом списке (добавлялся программно)
