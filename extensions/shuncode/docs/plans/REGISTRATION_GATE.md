# Стратегия Registration Gate для Shuncode AI

> Бизнес-план + техническая архитектура для выхода продукта в открытый доступ.
> Создано: 2026-02-13 | Статус: ✅ РЕАЛИЗОВАНО (2026-02-22/23)
>
> **Что реализовано:** shuncode-web (Next.js 16), Auth.js v5 (Credentials + Yandex + Google + Nodemailer),
> PostgreSQL (Prisma), лендинг, личный кабинет, админ-панель, система баннеров,
> защищённые скачивания, Docker-деплой на VPS (shuncode-ai.ru).
> Подробная документация: `shuncode-web/docs/`

---

## Контекст

- **Shuncode AI** — форк VS Code (не расширение), из-за View Zones для diff системы
- **Дистрибуция** — собранный installer/zip с сайта
- **Бэкенд** — Next.js (fullstack), сайт + документация + auth + бизнес-раздел
- **Своих моделей нет** — пользователи используют свои API ключи (OpenAI, Anthropic, OpenRouter, Ollama и т.д.)
- **В будущем** — возможен свой провайдер моделей через OpenRouter
- **Целевой рынок — РФ** — компании, которым Cursor недоступен

## 1. Зачем registration gate

**Проблема open access:** если просто выложить билд — нет контакта с пользователями, нет метрик, нет базы для монетизации.

**Что даёт gate (даже бесплатный):**
- Email-база пользователей (для рассылок, фидбека, upsell)
- Телеметрия привязанная к аккаунту (какие фичи используют, какие модели)
- Основа для будущих тарифов (Free / Pro / Enterprise)
- Легитимность для бизнес-клиентов (есть аккаунт = есть поддержка)
- Контроль пиратства (лицензионный ключ для корпоративных клиентов)

## 2. Разделение: что блокировать, что оставить

### Принцип: "попробуй бесплатно, залогинься чтобы работать серьёзно"

### БЕЗ регистрации (демо-режим)

| Функция | Доступ |
|---------|--------|
| Установка и запуск | Да |
| Настройка API провайдера (свой ключ) | Да |
| Базовый чат с AI | **Лимит ~20 сообщений** |
| Чтение файлов (read_file, list_files) | Да |
| Просмотр настроек | Да |
| Запись/редактирование файлов | Да (в рамках лимита) |
| Терминал | Да (в рамках лимита) |

Цель: пользователь видит, что продукт работает, и понимает ценность.

### ПОСЛЕ бесплатной регистрации

| Функция | Доступ |
|---------|--------|
| Безлимитный чат (с собственным API ключом) | Да |
| Все инструменты (write, edit, terminal, browser) | Да |
| Семантическая индексация / codebase search | Да |
| Cursor-like diff система | Да |
| Субагенты | Да |
| Все экспериментальные фичи | Да |

### Будущее (платный тариф)

| Функция | Тариф |
|---------|-------|
| Свои модели через OpenRouter (без своего API ключа) | Pro |
| Приоритетная поддержка | Pro |
| Enterprise фичи (SSO, audit log, remote config) | Enterprise |
| Повышенные лимиты | Pro/Enterprise |

### Почему именно такое разделение

- 20 демо-сообщений достаточно чтобы попробовать ВСЁ и понять ценность.
- Индексация, diff, субагенты — видны в настройках, но заблокированы. Это мотивирует зарегистрироваться.
- Регистрация бесплатная — барьер минимальный.

## 3. Техническая архитектура

```
Shuncode AI (клиент)                    shuncode.ai (бэкенд)
┌──────────────────┐               ┌──────────────────────┐
│                  │               │  Next.js fullstack    │
│  WelcomeView     │──register────>│  /api/auth/register   │
│  (кнопка)        │<──JWT token───│  /api/auth/login      │
│                  │               │  /api/license/verify   │
│  ShuncodeAuthProvider│──verify───────│                       │
│  (при запуске)   │<──features────│  PostgreSQL (Prisma)  │
│                  │               │  Auth.js (NextAuth)   │
│  SecretStorage   │               │                       │
│  (JWT кэш)       │               │  Яндекс ID, GitHub,  │
│                  │               │  Email (magic link)   │
└──────────────────┘               └──────────────────────┘
```

### 3.1 Бэкенд (Next.js)

**Auth:**
- Auth.js v5 (NextAuth) — готовая auth
- Провайдеры: Email (magic link) + GitHub + Google + **Яндекс ID**
- Яндекс ID критичен для РФ рынка (корпоративные пользователи)
- JWT с подписью — клиент проверяет подпись локально

**API endpoints:**
- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход
- `GET /api/license/verify` — проверка лицензии (при запуске)
- `POST /api/telemetry/event` — анонимная телеметрия (опционально)

**БД:**
- PostgreSQL (через Prisma ORM)
- Таблицы: `users`, `licenses`, `sessions`, `telemetry_events`
- Хостинг: Vercel + Supabase PostgreSQL (бесплатный тариф на старте)

### 3.2 Клиент (Shuncode AI) — ✅ РЕАЛИЗОВАНО

**Реализованная инфраструктура:**
- `src/config.ts` — URL-ы API (production: shuncode-ai.ru)
- `src/services/auth/providers/ShuncodeAuthProvider.ts` — auth (JWT, token exchange, refresh)
- `src/services/auth/AuthService.ts` — singleton, восстановление токена при запуске
- `webview-ui/src/context/ShuncodeAuthContext.tsx` — ShuncodeUser, subscribeToAuthStatusUpdate
- `webview-ui/src/components/account/AccountWelcomeView.tsx` — экран авторизации с баннером лимита
- `webview-ui/src/components/settings/ShuncodeAccountInfoCard.tsx` — карточка аккаунта

**Free-trial gate (реализован 2026-02-24):**
- `Controller.FREE_REQUEST_LIMIT = 20` — лимит сообщений без авторизации
- `Controller.checkFreeRequestGate()` — проверка лимита (вызывается из initTask, askResponse, interruptAndSend, sendMessage)
- `freeRequestCount` — персистентный счётчик в globalState
- При достижении лимита: VS Code notification + навигация на вкладку Account
- Баннер на стартовом экране чата для неавторизованных: «Добро пожаловать! У вас есть 20 сообщений...»
- Тот же текст на экране Account (AccountWelcomeView)
- i18n: RU + EN

**Проверка лицензии:**
- При запуске: `GET /api/license/verify` с JWT
- Кэшировать результат на 24 часа (работает offline)
- Если токена нет или expired — WelcomeView с предложением регистрации
- Premium фичи проверяют `isRegistered` через `ShuncodeAuthContext`

### 3.3 Offline-режим

Критично для РФ рынка (VPN, нестабильный интернет):
- JWT кэшируется в VS Code SecretStorage
- Проверка подписи JWT — локальная (публичный ключ вшит в клиент)
- Если нет интернета — работает по кэшированному состоянию
- Перепроверка при следующем онлайн-запуске

## 4. Gate в UI — ✅ РЕАЛИЗОВАНО

### Демо-режим (без регистрации)

- Баннер на стартовом экране чата: «👋 Добро пожаловать! У вас есть 20 сообщений для знакомства с Shuncode AI. Авторизуйтесь бесплатно, чтобы продолжить без ограничений.» + кнопка «Авторизоваться бесплатно»
- При достижении лимита: VS Code notification + автоматический переход на вкладку Account
- На экране Account: тот же текст про лимит + кнопка авторизации (OAuth через браузер)
- Все фичи доступны в рамках лимита (нет блокировки отдельных инструментов)

### Зарегистрированный пользователь

- Баннер не показывается
- Все фичи разблокированы, без лимита сообщений
- В навбаре: кнопка Account с профилем + logout

## 6. Конкурентное позиционирование (РФ рынок)

**Ключевые преимущества:**

| vs Cursor | vs Cline/Roo-Code (референс) |
|-----------|-------------------|
| Не требует подписки на западный сервис | Cursor-like diff система |
| Работает с любыми моделями (Ollama для on-premise) | Семантическая индексация |
| Русский интерфейс из коробки | Субагенты |
| Яндекс ID авторизация | Полноценный продукт (сайт, поддержка, доки) |
| Open source — можно аудировать | Профессиональный UI |

**Для компаний:**
- On-premise развёртывание (Ollama + свой сервер моделей)
- Код не уходит на внешние серверы (если Ollama)
- Аудируемый open source
- Российская юрисдикция
