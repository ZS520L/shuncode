## 1. DELETION (удаление строк)

### 1.1 Удаление одной строки
- [ ] Попросить AI удалить строку из файла
- [ ] View Zone (красная) появляется на правильной позиции
- [ ] Кнопки Accept/Reject видны внутри View Zone
- [ ] **Accept:** View Zone исчезает, строка остаётся удалённой
- [ ] **Reject:** View Zone исчезает, строка восстанавливается в файле
- [ ] Sidebar: запись появляется и исчезает после Accept/Reject

### 1.2 Удаление блока (3-5 строк)
- [ ] View Zone показывает все красные строки
- [ ] Accept/Reject работают корректно

### 1.3 Удаление первой строки файла (BUG 3 fix)
- [ ] **ВАЖНО:** Удалить первую строку (startLine=1)
- [ ] View Zone не крашится (lineAfter=0, не -1)
- [ ] Accept/Reject работают

## 2. ADDITION (добавление строк)

### 2.1 Добавление одной строки
- [ ] Зелёная строка появляется в файле
- [ ] View Zone с кнопками под зелёной строкой
- [ ] **Accept:** кнопки исчезают, зелёная подсветка убирается
- [ ] **Reject:** строка удаляется из файла, кнопки исчезают

### 2.2 Добавление блока кода (функция)
- [ ] Все зелёные строки подсвечены
- [ ] Кнопки под последней зелёной строкой
- [ ] Accept/Reject работают

### 2.3 Добавление в начало файла (BUG 3 fix)
- [ ] Добавить строки в самое начало файла
- [ ] Нет краша, позиции корректны

## 3. REPLACEMENT (замена)

### 3.1 Замена одной строки
- [ ] View Zone с красной строкой (старой) вверху
- [ ] Зелёная строка (новая) в файле
- [ ] View Zone с кнопками внизу после зелёной
- [ ] **Accept:** обе View Zones исчезают, зелёная остаётся
- [ ] **Reject:** зелёная удаляется, красная восстанавливается

### 3.2 Замена блока (3+ строк на 3+ строк)
- [ ] Все красные в View Zone, все зелёные в файле
- [ ] Accept/Reject работают

### 3.3 Замена с разным количеством строк (2 → 5, 5 → 2)
- [ ] Позиции корректны
- [ ] Accept/Reject работают

## 4. Множественные изменения в одном файле

### 4.1 Два изменения подряд
- [ ] Оба View Zone отображаются на правильных позициях
- [ ] Accept первого не ломает второй
- [ ] Reject первого корректно сдвигает второй (BUG 4 fix)

### 4.2 Reject снизу вверх
- [ ] Reject нижнего изменения не ломает верхнее
- [ ] Позиции пересчитываются (PositionTracker)

### 4.3 Reject сверху вниз
- [ ] Reject верхнего изменения → View Zone нижнего пересоздаётся на правильной позиции (BUG 4 fix)

### 4.4 Accept All / Reject All для файла (BUG 2 fix)
- [ ] Accept All: все View Zones исчезают
- [ ] Sidebar: все записи для файла удалены
- [ ] Reject All: все изменения откатываются
- [ ] Sidebar: все записи для файла удалены

## 8. System Edit Detection (BUG 1 fix)

### 8.1 DiffReverter не вызывает phantom recalculate
- [ ] Открыть Developer Console (Help → Toggle Developer Tools)
- [ ] Сделать Reject на изменении
- [ ] В консоли НЕ должно быть `[DiffSystem] Manual edit detected` после Reject
- [ ] Если есть — BUG 1 не исправлен

### 8.2 DiffApplier не вызывает phantom recalculate
- [ ] Сделать новое изменение через AI
- [ ] В консоли НЕ должно быть `[DiffSystem] Manual edit detected` при применении

## 9. Edge Cases

### 9.1 Пустой файл
- [ ] AI пишет в пустой файл → addition работает
- [ ] Accept/Reject работают

### 9.2 Файл с одной строкой
- [ ] AI заменяет единственную строку → replacement работает
- [ ] AI удаляет единственную строку → deletion работает

### 9.3 Очень большой файл (1000+ строк)
- [ ] Изменение в середине — позиция View Zone корректна
- [ ] Нет заметного lag при создании View Zone

### 9.4 showDiffVisualization (BUG 9 fix)
- [ ] showDiffVisualization создаёт полноценные View Zones С кнопками Accept/Reject
- [ ] Diff регистрируется в StateStorage, CheckpointStorage, PendingChangesStorage
- [ ] Accept/Reject работают (не orphan zones)
- [ ] Нет ошибок в консоли
