## 1. Информация о проекте

### Название проекта

**Shuncode AI — интеллектуальная среда разработки программного обеспечения на базе технологий искусственного интеллекта**

### Краткое описание (аннотация)

Shuncode AI — это интегрированная среда разработки (IDE) на базе Visual Studio Code с встроенным ИИ-ассистентом, способным автономно выполнять задачи разработки ПО: редактирование файлов, выполнение команд терминала, поиск по кодовой базе, навигация по вебу и взаимодействие с внешними инструментами.

В отличие от существующих зарубежных аналогов (Cursor, Windsurf, GitHub Copilot), Shuncode AI:
- Ориентирован на российский рынок с локализацией и оплатой в рублях
- Предоставляет доступ к передовым языковым моделям (Claude, GPT, Gemini) через единый прокси-сервис
- Реализует визуальную систему контроля изменений (inline diff) для безопасной работы с ИИ
- Поддерживает протокол MCP (Model Context Protocol) для интеграции с произвольными внешними сервисами

### Направление (лот)

**Фундаментальные языковые и мультимодальные модели ИИ** — разработка инструментов и интерфейсов для практического применения LLM в профессиональной деятельности разработчиков ПО.

## 2. Проблема

### Описание проблемы

Российские разработчики программного обеспечения сталкиваются с тремя критическими проблемами при использовании ИИ-инструментов для кодирования:

**1. Недоступность лидирующих решений**
- Cursor ($20/мес) — не принимает оплату из РФ
- GitHub Copilot — ограничен санкциями для ряда организаций
- Windsurf — аналогичные ограничения по оплате
- Ни один из этих продуктов не имеет русской локализации

**2. Высокий порог входа**
- Для использования LLM-моделей необходимо самостоятельно получать API-ключи у зарубежных провайдеров
- Оплата API требует иностранных карт или криптовалюты
- Настройка и интеграция моделей требует технических знаний

**3. Отсутствие контроля над ИИ-агентом**
- Существующие решения либо блокируют каждое действие ИИ (неудобно), либо дают полный доступ (небезопасно)
- Нет промежуточного варианта с визуальным контролем post-factum

### Целевая аудитория

- **Первичная:** Российские разработчики ПО (оценка: 1.5-2 млн человек)
- **Вторичная:** IT-компании РФ, ищущие инструменты повышения продуктивности
- **Третичная:** Разработчики из СНГ (русскоязычная аудитория)

### Объём рынка

- **TAM (Total Addressable Market):** Глобальный рынок ИИ-инструментов для разработки — $15-20 млрд к 2027
- **SAM (Serviceable Addressable Market):** Российские разработчики, готовые платить за IDE — ~500K человек × $10/мес = $60 млн/год
- **SOM (Serviceable Obtainable Market):** 1% SAM в первый год = 5000 пользователей × $10/мес = $600K/год (~55 млн руб/год)

## 3. Решение

### Продукт

Shuncode AI — desktop-приложение (IDE) для Windows, macOS, Linux. Базируется на собственном форке Visual Studio Code с интегрированным ИИ-расширением.

### Ключевые технологии ИИ

| Технология | Описание | Статус |
|-----------|----------|--------|
| Мультимодельная маршрутизация | Подключение к 40+ LLM (Claude, GPT, Gemini, DeepSeek и др.) через единый API | ✅ Реализовано |
| Агентный цикл выполнения | ИИ автономно планирует и выполняет цепочки действий (чтение → анализ → редактирование → проверка) | ✅ Реализовано |
| Inline Diff система | Визуализация изменений прямо в редакторе с кнопками Accept/Reject | ✅ Реализовано |
| Model Context Protocol (MCP) | Подключение произвольных внешних инструментов и источников данных | ✅ Реализовано |
| Tree-sitter валидация | Синтаксическая проверка сгенерированного кода для 16 языков | ✅ Реализовано |
| Контекстное управление | Автоматическое сжатие и приоритизация контекста для оптимального использования контекстного окна модели | ✅ Реализовано |

### Доступные ИИ-инструменты (18 штук)

1. **Чтение файлов** — анализ существующего кода
2. **Редактирование файлов** — модификация кода с визуальным diff
3. **Создание файлов** — генерация нового кода
4. **Применение патчей** — массовые изменения через unified diff
5. **Выполнение команд** — терминальные операции (сборка, тесты, git)
6. **Поиск по файлам** — regex-поиск по кодовой базе (ripgrep)
7. **Листинг файлов** — навигация по структуре проекта
8. **Определения кода** — поиск классов, функций, интерфейсов
9. **Веб-поиск** — поиск информации в интернете
10. **Загрузка URL** — чтение документации и веб-страниц
11. **Браузер** — визуальное взаимодействие со страницами (Puppeteer)
12. **Диагностика** — чтение ошибок линтера/компилятора
13. **MCP инструменты** — произвольные внешние инструменты
14. **Подзадачи** — декомпозиция сложных задач
15. **Сжатие контекста** — оптимизация длинных диалогов
16. **Планирование** — режим анализа без модификаций
17. **Правила** — создание пользовательских правил для ИИ
18. **Отчёты об ошибках** — автоматическая фиксация проблем

### Текущий уровень разработки (TRL)

**TRL 6-7** — Продукт разработан и прошёл тестирование в реальных условиях разработки ПО. Основные компоненты реализованы и функционируют.

- Поддержка 40+ провайдеров LLM
- React UI (232+ компонентов)
- gRPC/Protobuf коммуникация между процессами
- Система чекпоинтов и откатов

**Квартал 2 (месяцы 4-6): Улучшение ИИ-компонентов**
